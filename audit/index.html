Guide de l'audit

Pourquoi cet audit

Pour chaque ICPE, Géorisques liste à la fois une adresse postale et des coordonnées (latitude/longitude). Quand les deux ne concordent pas, la carte peut être trompeuse — surtout quand l'écart fait basculer la réponse à « ce site est-il dans une réserve naturelle ? ». Cet outil te permet de trier les désaccords un par un.

Les trois groupes

Réserves naturelles
Sites où l'adresse et les coordonnées divergent sur l'appartenance à une réserve, ou en sont très proches. À traiter en priorité.
Grands écarts
Distance ≥ 500 m, commune incorrecte, adresse irrésoluble, ou coordonnées structurellement invalides.
Petits écarts
Distance entre 25 m et 500 m, ou cas où le reverse confirme déjà la commune. Triage rapide possible.

Comment revoir un site

  1. Lis l'identité du site (nom, commune, activité) dans le panneau de droite.
  2. Regarde la mini-carte. Le point rouille = position enregistrée. Le point ocre = adresse géocodée. La ligne grise pointillée relie les deux.
  3. Bascule sur la couche « Ortho » si tu veux voir le bâtiment réel.
  4. Compare avec l'adresse inscrite (rue, parc, zone industrielle).
  5. Choisis un verdict (voir ci-dessous).
  6. Ajoute une note si tu hésites ou si quelque chose mérite d'être marqué pour l'enquête.
  7. Coche « Pertinent pour l'enquête réserves » si l'écart change la réponse à « dans une réserve ? ».
  8. Clique « Enregistrer & suivant » (ou tape Entrée).

Les quatre verdicts

1 Garder les coordonnées enregistrées
L'écart est cosmétique. Exemple : la raffinerie a son adresse au portail mais le point enregistré est à la torchère. Même site.
2 Utiliser l'adresse géocodée
Les coordonnées enregistrées sont visiblement fausses ; l'adresse est la source de vérité.
3 Placer le point manuellement
Les deux sont faux ; tu peux voir le bâtiment réel sur la couche ortho. Tu cliques sur la carte pour placer un point.
4 À vérifier sur le terrain
Impossible de décider depuis la documentation ; à inspecter pendant la semaine d'enquête.

Exporter et partager son travail

  1. Quand tous les sites d'un bucket sont revus, clique sur « Exporter le brouillon ».
  2. Télécharge le fichier JSON et dépose-le dans données-georisques/audit/coordonnees-audit-reviews/.
  3. Commit et push : git add … && git commit -m "review bucket-…" && git push
  4. Actualise cette page : le bucket apparaît avec une coche ✓ et tes initiales.

Après l'audit : appliquer les corrections

Une fois que suffisamment de buckets ont été revus et commités :

  1. Lance python3 scripts/apply_corrections.py — ce script lit les fichiers JSON de revue commités et produit coordonnees-corrections.csv, un tableau complet des décisions prises par les enquêteurs.
  2. Vérifie les décisions dans le tableau des décisions avant de les appliquer.
  3. Relance python3 scripts/enrichir_libelles.py — le script lit automatiquement le sidecar de corrections et met à jour les coordonnées dans le CSV de la carte.
  4. Commit et push — la carte interactive affiche désormais les coordonnées corrigées.

Le script est idempotent : le relancer avec les mêmes revues produit le même résultat. Pour prévisualiser sans écrire : python3 scripts/apply_corrections.py --dry-run.

Travailler en équipe

La coordination se fait à l'oral : « je prends le bucket reserves-01 ». Pas besoin de réservation formelle — la page lit le dossier de revues sur GitHub à chaque chargement et sait toujours qui a quoi terminé. Si deux personnes soumettent le même bucket, le commit le plus récent gagne.

En cas de problème

  • Le bucket est marqué ⚠ audit plus récent : l'audit a été relancé depuis ta revue. Clique « Réimporter les décisions » pour migrer ce que tu as déjà fait.
  • Le bouton Actualiser affiche « Limite atteinte » : GitHub limite les requêtes anonymes à 60/heure. Attends quelques minutes.
  • La mini-carte est vide : le site a des coordonnées invalides (null_island). Le panneau de droite donne le détail.
